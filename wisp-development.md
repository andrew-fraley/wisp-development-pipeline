**Security Audit & WISP Development**

**Objective**
The focus of this section of the workflow pipeline is to translate the intake data from the questionnaire and intake consultation into a custom tailored IRS WISP.

**Process**
1. Security Audit
- Review the intake consultation notes and questionnaire
- Identify:
	- Security Gaps
	- Operational Risks
	- Missing Security Controls
- Take findings and draft the Security Audit Findings (document) and use this as the reference along with the intake data to create the Written Information Security Plan (including policies and procedures to implement to harden the overal security posture).

2. WISP Development
- Build a custom WISP covering the following:
	- IRS WISP compliance & FTC Safeguard Rules (industry expectations)
	- Security controls addressing gaps found in the security audit findings
	- Security plan that aligns with business operations

**Core WISP Componenets**
- Data & Asset Inventory
- Roles & Responsibilities
- Risk Assessment Summary
- Adminstrative Safeguards
- Technical Safeguards
- Physical Safeguards
- Incident Response Plan
- Ongoing Monitoring & Review
- Additional Attachements (example: Record Retention Policy, Employee/Contractor Acknowledgement of Understanding, etc.)
Note: This is the basic template for my workflow but does not exhaust the overall scope every WISP will cover, each WISP is tailored custom for the unique client, business operations, and overall organization structure.
