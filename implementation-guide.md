**Implementation Guide**

**Overview**

This is a multi-function step in the pipeline. It serves as the frame for theimplementation consultation session that the information security professional will lead the client through the whats, hows, and whys of implementing the IRS WISP, along with any security controls necessary to align with the client's new information security plan. 

The second thing the implementation step accomplishes is the actual Implementation Strategy Guide delivered to the client. I have two templates that are used in this step, I have an overall consulation meeting guide for the consultation session with the client, this simply acts as a structure for the consultant to streamline the actual session and cover all relevant topics in guiding the client through their own implementation guide document. And the second template is the client's deliverable Immplementation Strategy Guide document.

**Consulation Process**

**1. Opening:** 
  - This reiterates the scope of the WISP project with the client.

**2. WISP Walkthrough:** 
  - This goes through the actual draft of the WISP document with the client. 
  - This may lead to small changes to the document if necessary, but the WISP at this step should be in its deliverable state. (The consultant should already reviewed and edited the document based on the project scope, client intake information, and any other relevant areas relating to the WISP docuemnt).

**3. Gap Identification**
	- This section lets the consultant go through all identified security gaps identified in the security audit.
	- What security controls are in place already, what is lacking or partially implemented.
	- This goes through the security audit and guides the client through identified security controls needed to strengthen the client's security posture and adhere to the newly created (or reviewed) IRS WISP.

**4. Prioritization**
	- This goees through the necessary controls and security measures they need to implement based on the findings and WISP.
	- This ranks the priority of each implementation measure, and a guide on how to properly implement them and suggested timelines and deadlines for when these controls should be in place.

**Closeout**
	- Recap key points and reinforce any high priority risks and security matters. (Make sure the client understands how to utilize the actual Implementation Strategy Guide document.)
	- Advise on any additional services, reviews, audits, consulations, etc.
	- Agree on the final deliverable timelines and when and what to expect as you close out the client's project.
