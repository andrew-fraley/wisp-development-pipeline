**Security Audit**

**Overview**

The security audit/risk assessment is based on the NIST CSF and RMF (cybersecurity and risk managment frameworks). The audit is conducted around the provided information obtained in the intake process of the client (questionnaire and intake interview/consultation) and the scope of the project.

**Purpose**

The reason for a security audit is the compliance requirements and the entire WISP and security measures custom tailored for the client are derived from the findings in the security audit. What security practices are in place, what is lacking, are there half measures that need to be completed, these are some of the questions the security audit findings will answer. This allows the security professional to develop an adequate data security compliant WISP, and beyond that it should allow for an in depth implementation guide for the client, under the direction of the infosec professional and the data security coordinator, to mitigate risks to the data security of their practice.

**Conclusion**

The audit conducted for the client is integral to the WISP development pipeline and is also required for compliance. Beyond this, I utilize my security audit findings to develop the WISP and my implmentation guide for the client. So, as an important part of the process, part of the final deliverable of a project, I publish a Security Audit Findings docuement to deliver to the client to accompany their WISP and implementation guide. It is heavily referenced in the implementation guide document and the implementation consultation.
